Statistics

Enabling this feature will send the following information to the developer:
  • Browser version (so I know if I can use new features or should maintain compatibility with older versions)
  • Extension version (so I know when autoupdate works)
  • Size of cache (Cache is a memory that keeps location information for sites that you've visited. This number shows how many records does it have. Knowing it tells me if I should optimize related part of extension)
  • Usage of precache (Precache is a location information for most popular sites, collected while building the extension. This number shows how many sites from this list you've visited. Knowing it tells me if it's useful at all)
  • Usage of "top 100 sites" part of precache (Like previous, but only first 100 sites from precache are accounted. Knowing it tells me what happens if I make precahe 10 times smaller)
  • Usage of "extraHosts" part of precache (Like two above, but uses list of sites which once were among most popular requests to flag-button.tk. Knowing it tells me if I should consider minimizing it)
  • Amount of requests to source (How many times this extension requested location information for a new site. Knowing it tells me if I should optimize related part of extension)
  • Your current browser language (so I know if I need to translate the extension to more languages)
  • All preferences except those that might have personal information (ipinfodb.com access key, links list and custom icons - these preferences won't be sent. Knowing how users change preferences will give me hints how settings should be configured by default)
Knowing how you are using this extension will motivate the developer to fine-tune it for your needs!
